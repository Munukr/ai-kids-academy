# AI Kids Academy

A production-ready Flutter educational app for children aged 5–7 that teaches AI fundamentals through 12 interactive lessons with mascot robot Beep/Бип/ביפ. Fully localized in English, Russian, and Hebrew with complete RTL support.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 5000)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string

### Flutter App (ai-kids-academy/)
- `cd ai-kids-academy && flutter pub get` — install dependencies
- `cd ai-kids-academy && flutter run` — run on device/emulator
- `cd ai-kids-academy && flutter test` — run tests
- `cd ai-kids-academy && flutter build apk --debug` — build debug APK

## Stack

### Monorepo (pnpm)
- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

### Flutter App
- Flutter 3.22+ / Dart 3
- Provider — state management
- SharedPreferences — local progress storage
- Google Fonts (Nunito) — typography
- Flutter Localizations — RTL/i18n

## Where things live

- `ai-kids-academy/` — Flutter Android app
  - `lib/` — Dart source code
  - `assets/data/` — lesson JSON files (EN/RU/HE)
  - `.github/workflows/build-apk.yml` — CI/CD for APK builds
  - `README.md` — detailed architecture docs
- `artifacts/api-server/` — Express API backend
- `lib/api-spec/` — OpenAPI contract
- `lib/api-client-react/` — generated React Query hooks
- `lib/api-zod/` — generated Zod schemas

## Architecture decisions

- All UI strings live in `lib/constants/app_strings.dart` — single source of truth for all 3 languages, no ARB files
- Lessons loaded from local JSON assets with in-memory cache — ready for future remote URL override
- Progress stored in SharedPreferences as JSON — simple, reliable, no DB required
- Lessons unlock sequentially (complete lesson N to unlock lesson N+1)
- RTL handled via `Directionality` widget per-screen, locale set on MaterialApp

## Product

12 lessons teaching children aged 5–7 the basics of AI: what it is, how to ask good questions (prompts), image generation, story generation, safe homework help, online safety, fact checking, creativity, voice commands, photo analysis, and a final mission. Each lesson has multi-page content with mascot commentary and a 3-question quiz with a star reward on completion.

## User preferences

- Flutter app is the primary deliverable
- No Firebase, no login, no payments, no paid services
- All lesson content must exist in all 3 language JSON files
- GitHub Actions workflow must build and upload APK as artifact

## Gotchas

- The Flutter project lives in `ai-kids-academy/` — run Flutter commands from inside that directory
- `android/local.properties` is gitignored and generated by Flutter toolchain during build
- The GitHub Actions workflow installs Flutter — the APK artifact is available in GitHub Actions after each push
- Hebrew text is RTL — each screen wraps content in `Directionality(textDirection: isRtl ? TextDirection.rtl : TextDirection.ltr, ...)`
